# 案例走查：一次完整的 AlphaLoop 闭环

下面用一个真实发生过的核验场景，演示这套模式怎么跑。标的代码已脱敏（用占位符代替），重点看**流程和纪律**，不构成任何投资建议。

## 场景：「重稀土卡日本」反制叙事

### 第一步 · 交易员出框架

交易员转来一篇文章 + 另一个 AI 给的结论，抽象出的因果链是：

> 中日关系螺旋恶化 → 中国收紧中重稀土出口管制 → 卡住日本半导体 / AI / MLCC 的原料 → 氧化镝、氧化钇、氧化锆「三神材」受益，附一串受益标的。

这是典型的「从市场变化（材料涨价刷屏）抽象出逻辑（反制叙事）」并交给 Agent。注意：交易员给的是**方向**，证据和选股交给 Agent。

### 第二步 · Agent 验证执行

Agent 先把这篇文章 **降级为 C 级假设**，然后逐条对应流程图五步：

| 流程环节 | 这次实际做的事 | 结果 |
|---|---|---|
| 拆解 & 定级 | 拆出「镝/钇是 MLCC 掺杂剂」「中国管制了镝/钇」「氧化锆是稀土」「涨价 140 倍」等原子声明 | 4 条关键声明待查 |
| 溯源验证 | 找一手源：出口管制清单、行业报价机构（Argus，路透引用）的氧化钇报价 | 核到「140 倍是真的，但是**海外价**，国内只涨 70% 量级」|
| 产业链分析 | 套 `strategic-materials` 五层漏斗 | **揪出关键失真**：文章把「氧化锆」当稀土，其实锆是搭便车的国产替代腿，和镝/钇的反制逻辑是两条腿 |
| 实时行情选股 | 用 `stock-data-fetch` 拉最新收盘价 | 资源端集体涨停；同时校正一个个股代码错误、补漏一个遗漏标的 |
| 输出结论 | 打 ✅🔴 标签 + 标的分档 | 出核验矩阵，回写 openorder wiki |

核验矩阵（节选，示意）：

| 声明 | 判定 | 一手源 / 说明 |
|---|---|---|
| 镝/钇是 MLCC 稀土掺杂剂 | ✅ | 标准文本 + 行业资料确认 |
| 中国已把镝/钇列入出口管制 | ✅ | 出口管制清单 |
| 氧化锆属于稀土 | 🔴 | 锆不是稀土；它是另一条「国产替代」腿，逻辑要拆开 |
| 氧化钇一年涨 140 倍 | 🟡 | 真，但那是**海外/超高纯价**；国内价涨幅小得多，矿企吃不到这个价差 |

### 第三步 · 交还交易员决策

Agent 给的一句话落点：

> 要买先想清楚买哪条腿——**反制叙事看重稀土，国产替代看锆粉**；那 140 倍是海外超高纯价，国内矿企吃不到这个价差。

仓位、择时、是否买入——扳机留给交易员。

## 这个案例说明了什么

1. **人给的框架骨架是真的**（反制逻辑成立），但 Agent 溯源后揪出两处失真——「氧化锆混入稀土」「140 倍是海外价」。**若不验证直接采信，选股方向就会偏。** 这正是三条铁律的价值。
2. **闭环里唯一没走完的是反馈环**：交易员后续是否成交、结果如何，需要回流给 Agent（用 `trade-journal` 记来源框架 + openorder 做盈亏归因），系统才会越来越懂这位交易员的判断模式。

## 复用这套走查

把这个案例当模板：任何"有人转给我一段投资逻辑，让我判断真假并选股"的场景，都可以照这五步跑——拆解 → 溯源 → 产业链 → 选股 → 出结论 → 交还决策。
