# 换成你自己的领域

AlphaLoop 的分工模式（人出方向 → Agent 验证执行 → 人决策 + 反馈闭环）并不绑定股票投资。任何"人凭经验判断方向、机器负责查证落地"的领域都能套用。本文讲怎么 fork。

## 哪些部分是领域无关的（直接复用）

这几层换领域时基本不用改，它们是通用纪律：

- **`claim-verification`** —— "二手信息默认是假设、逐条溯源到一手"这套方法论适用于任何需要事实核查的领域（尽调、法律、学术、新闻）。
- **`agent-tool-escalation`** —— 工具失败升级、修正已有信息前先验证，跟领域无关。
- **`openorder`** 的三层架构（Raw / Wiki / Schema）+ `log.md` 时间线 + Lint 操作 —— 是通用的"AI 维护的知识库"骨架，换领域只改触发词和模板。
- **`trade-journal`** 的"建仓即记 + 来源框架归因"思路 —— 任何"决策 → 结果 → 归因"的闭环都能借用（把"交易"换成你领域的"决策动作"）。

## 哪些部分要替换成你的领域

### 1. 编排器 `SKILL.md` 的路由表和触发词

把根目录 `SKILL.md` 里的：
- `description`（触发场景）换成你领域的关键词
- 路由表换成"你领域的信号 → 调度哪个 skill"

### 2. 示例框架层 `strategic-materials` → 你自己的分析框架

`strategic-materials` 是「人交给 Agent 的框架」的范例——它演示了一个领域框架如何：
- 开头声明自己是**编排器**，强制接线验真 + 取数 + 落档；
- 用"几层漏斗 + 每层自检问题"把人的判断结构化；
- 明确**边界**（什么情况下不该用这个框架）。

照这个结构写你自己的框架即可。比如：
- **加密研究** → "代币经济 → 协议护城河 → 链上数据 → 叙事周期"漏斗
- **生物医药** → "靶点 → 临床进度 → 竞争格局 → 现金跑道"漏斗
- **房产** → "区位 → 供需 → 政策 → 租售比"漏斗

### 3. 行情/数据工具 `stock-data-fetch` → 你领域的数据源

换成你领域的实时数据获取方式（链上 API、临床数据库、房产成交数据等），保留"多源兜底 + 标时间和源"的纪律。

### 4. `openorder` 的模板和触发词

按 [`skills/openorder/docs/customize.md`](../skills/openorder/docs/customize.md) 把公司档模板换成你领域的实体（药物管线 / 协议 / 房产档案），触发词换成你领域的关键词。

## 改完后

1. 重新跑 `./install.sh` 让软链生效。
2. 开新对话测一下：给一段你领域的"逻辑框架"，看 AI 是否按"假设 → 验证 → 落地 → 交还决策"跑。
3. 三条铁律保持不变——它们是这套模式之所以可靠的根本。
